# 正矿供应链门户 · 从零到一设计全案 (Pre-Start Design Proposal)

> **设计背景**：在代码起步前，我们需要一份能够指导全局的“施工图纸”，确保最终产出的网站具备工业级专业感、极高的可维护性、以及数据驱动的灵活性。

---

## 第一阶段：核心愿景与产品定调 (Mindset)

在写第一行代码前，我们必须明确网站的“灵魂”：

1. **项目核心定位**：
   - **不是**一个简单的文字展示站。
   - **而是**一个工业级、仪表盘化的“产业互联网入口”。
   - **关键词**：专业 (Professional)、工业 (Industrial)、透明 (Transparent)、智能 (Agentic)。

2. **设计预期达成效果**：
   - 用户打开网页第一秒的感觉：**“这是一家掌握核心数据的供应链巨头”**。
   - 实现方式：硬朗的线条、深邃的蓝灰色调、实时更新的指标感。

---

## 第二阶段：视觉系统设计 (Visual DNA)

我们要在 CSS 编写前定死这套“视觉宪法”：

### 1. 颜色令牌 (Color Tokens)
- **主品牌色 (Primary)**: `#2563eb` (Blue-600) —— 代表科技、安全与信任。
- **底色 (Base)**: `#f8fafc` (Slate-50) —— 营造高级感的微灰背景，非纯白，减少疲劳感。
- **状态色 (Status)**: 
  - `Emerald-500` 代表“稳健/看涨”。
  - `Rose-500` 代表“风险/预警”。

### 2. 布局逻辑 (Layout Engine)
- **容器协议**：
  - **Standard**: `max-w-7xl (1280px)` 定死全局最大宽度，防止内容在大屏上散乱，保持紧凑感。
  - **Prose**: `max-w-4xl (896px)` 定死研报/正文宽度，追求极致的阅读舒适度。
- **深度质感**：
  - 导航栏必须有 `backdrop-blur` 模糊效果，体现层次感。
  - 卡片必须有微弱的 `hover:-translate-y-1` 位移，让页面“活”起来。

---

## 第三阶段：架构选型与数据哲学 (Technical Architecture)

为了保证“我改需求你只需改配置”，我们需要一套“配置驱动”的架构：

1. **技术栈选型**：
   - **React + Vite**：保证开发的极速反馈。
   - **Tailwind CSS v4**：利用效能工具直接定义视觉，避免冗长的 CSS 维护。
   - **Hash Routing**: 确保在任何 Nginx 环境下点击即用，零配置部署。

2. **数据哲学 (Key Concept)**：
   - **“UI 只是外壳，数据才是核心”**。
   - **硬性规定**：所有**文案、价格、名单、研报标题**严禁写死在代码里。必须全部抽取到 `src/data/*.json` 中。
   - **效果**：未来修改内容，只需改 JSON，不需要触碰复杂的业务逻辑。

---

## 第四阶段：后端与运维中枢设计 (Operations)

网站不是孤岛，它需要和 AI 指挥部并联：

1. **容器化编排 (Isolation)**：
   - 物理隔离：网关 (Nginx)、官网 (Website)、AI大脑 (Dashboard)、回调站 (AI-bot) 各自独立。
2. **自动化链路 (The Loop)**：
   - **同步逻辑**：Obsidian (知识源) ➡️ 执行脚本 ➡️ 更新代码库 ➡️ 自动部署。
   - **部署指令缓存**：在 `03_Operations_Hub` 预置一键同步脚本，确保“本地敲代码，云端秒生效”。

---

## 第五阶段：实施路线图 (Implementation Roadmap)

1. **Day 1**: 初始化 Vite 项目，配置 Tailwind 主色调令牌与全局 `1280px` 布局。
2. **Day 2**: 构建核心 UI 原子（Navbar, Footer, Card），实现高度的一致性。
3. **Day 3**: 开发数据解析层，创建 `siteConfig.js` 和业务 JSON 模板。
4. **Day 4**: 串联页面逻辑，引入懒加载与错误隔离 (ErrorBoundary)。
5. **Day 5**: 编写 Docker 编排配置，完成云端 Nginx 路由映射。

---

## 总结：为什么这样设计能“一步到位”？

如果我们在开工前就确定了：**“1280px 宽度 + 蓝色调 + JSON 数据驱动 + Nginx 网关中枢”** 这套组合拳，那么后期你只需要跟我说：
- *“把 siteConfig 的地址改了”*
- *“在 JSON 里加几个矿样价格”*

**我们跳过了“猜你想做什么”的阶段，直接进入了“参数化执行”的工业化协作模式。** 这就是这份“反向推理全案”的核心价值。
